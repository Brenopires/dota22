**In MatchStateMachine:**

1. **Add endByTowerDestruction method:**
```typescript
endByTowerDestruction(destroyedTowerTeam: Team): void {
  if (this.phase === MatchPhase.ENDED) return;
  this.transition(MatchPhase.ENDED);
  // No score change needed -- tower destruction is a separate win condition
}
```
This uses the existing `transition()` guard which prevents backward transitions. The transition emits MATCH_STATE_CHANGE with phase: ENDED, which triggers BattleScene.onMatchStateChange -> endMatch(). No special event needed -- the existing pipeline handles it.

2. **Filter onKill to only count hero kills:**
The existing onKill handler increments score on EVERY HERO_KILLED event. Since BossEntity and TowerEntity override die() to emit their own events (BOSS_KILLED, TOWER_DESTROYED) instead of HERO_KILLED, this should already work correctly.

However, as a safety net, add an entityType check to the onKill handler:
```typescript
private onKill({ victim, killerId }: { victim: BaseEntity; killerId?: string }): void {
  if (this.phase === MatchPhase.ENDED) return;
  // Only count hero kills for score (boss/tower have their own events)
  if (victim.entityType !== 'hero') return;
  if (victim.team === Team.A) this.score.teamB++;
  else if (victim.team === Team.B) this.score.teamA++;
  EventBus.emit(Events.SCORE_UPDATED, { ...this.score });
}
```
Note: Also update the team check to use `else if` to avoid scoring for Team.NEUTRAL entities if they ever hit this handler.

3. **Import Team.NEUTRAL awareness:**
The existing onKill uses `victim.team === Team.A` -- if a NEUTRAL entity somehow reaches this handler, neither branch fires, so score is not affected. The `entityType` check above is the belt-and-suspenders approach.
