<task type="auto">
  <name>Task 2: Create BossEntity class</name>
  <files>src/entities/BossEntity.ts</files>
  <action>
Create `src/entities/BossEntity.ts` extending BaseEntity. Follow the Hero constructor pattern for physics/visual setup.

**Class structure:**
- `readonly entityType = 'boss' as const`
- `phase: BossPhase = BossPhase.NORMAL`
- Private fields: `baseMaxHP`, `baseDamage`, `minutesElapsed = 0`, `attackTimer = 0`
- Phase thresholds: enraged at BOSS_ENRAGED_THRESHOLD (0.6), dying at BOSS_DYING_THRESHOLD (0.25)

**Constructor (scene, x, y):**
- Call `super(scene, x, y, BOSS_BASE_HP, Team.NEUTRAL)`
- Store `baseMaxHP = BOSS_BASE_HP`, `baseDamage = BOSS_BASE_DAMAGE`
- Create visuals as Container children (following Hero pattern):
  - A large circle (BOSS_RADIUS) with dark red/purple color (0x8B0000)
  - An outer pulsing ring (BOSS_RADIUS + 8) with 0xff4444 stroke, animated with scene.tweens
  - A "B" label text centered on the entity (monospace, bold, 16px, white)
  - An inner glow circle with ADD blend mode, subtle pulse tween
- Add to scene: `scene.add.existing(this)` then `scene.physics.add.existing(this)`
- Configure physics body: `setCircle(BOSS_RADIUS, -BOSS_RADIUS, -BOSS_RADIUS)`, `setCollideWorldBounds(true)`, `setBounce(0)`, `setDrag(400)`, `setMaxVelocity(BOSS_MOVE_SPEED)`

**Public methods:**
- `scalePower(minute: number): void` -- Updates minutesElapsed. Calculates multiplier = `1 + (minute * BOSS_SCALING_PER_MINUTE)`. Updates maxHP = `Math.floor(baseMaxHP * multiplier)`. Heals ONLY the HP delta from the new scaling (NOT full heal). Updates internal baseDamage tracking for getAttackDamage().
- `getAttackDamage(): number` -- Returns `Math.floor(baseDamage * (1 + minutesElapsed * BOSS_SCALING_PER_MINUTE))` plus sum of STAT_BUFF buffs.
- `getAttackInterval(): number` -- Returns interval based on current phase: NORMAL -> BOSS_ATTACK_INTERVAL, ENRAGED -> BOSS_ENRAGED_ATTACK_INTERVAL, DYING -> BOSS_DYING_ATTACK_INTERVAL.
- `updateBoss(dt: number): void` -- Calls `this.updateBuffs(dt)`, decrements attackTimer by dt.
- `getUniqueId(): string` -- Returns `'boss_neutral'`.
- `getArmor(): number` -- Returns `BOSS_BASE_ARMOR + this.minutesElapsed` (armor increases each minute).

**Override takeDamage:**
- Call `super.takeDamage(rawDamage, sourceId)` to get finalDamage.
- If finalDamage > 0, call `this.checkPhaseTransition()` and show a damage number (simpler than Hero -- just a red floating text that fades).
- Return finalDamage.

**Private checkPhaseTransition():**
- Calculate hpRatio = currentHP / maxHP.
- If hpRatio <= BOSS_DYING_THRESHOLD AND phase !== DYING: set phase = DYING, emit BOSS_PHASE_CHANGED with `{ phase: BossPhase.DYING, boss: this }`, call `showPhaseTransitionVFX()`.
- Else if hpRatio <= BOSS_ENRAGED_THRESHOLD AND phase === NORMAL: set phase = ENRAGED, emit BOSS_PHASE_CHANGED, call `showPhaseTransitionVFX()`.
- IMPORTANT: Check DYING first (lower threshold) to handle case where boss takes massive damage skipping ENRAGED.

**Private showPhaseTransitionVFX():**
- Screen shake via `(this.scene as any).vfxManager?.directionalShake(0, 0.01, 500)` if available.
- Flash large text ("ENRAGED!" or "DYING!") at boss position, yellow/red color, fades out over 1.5s.
- Change the visual circle tint based on phase (NORMAL: 0x8B0000, ENRAGED: 0xFF4500, DYING: 0xFF0000).

**Override die(killerId):**
- CRITICAL: Do NOT call super.die() -- it emits HERO_KILLED which would trigger hero kill scoring, respawn, XP.
- Instead, duplicate the idempotent guard: `if (!this.isAlive) return;`
- Set `this.isAlive = false`, disable physics body (velocity 0, setEnable(false), setCircle(0)).
- Emit `Events.BOSS_KILLED` with `{ victim: this, killerId }`.
- Call `this.onDeath(killerId)`.

**Protected onDeath(killerId):**
- VFX: large death explosion via vfxManager.spawnDeathExplosion if available.
- Tween: shrink + fade + rotate over 800ms, then setVisible(false).

**Do NOT:**
- Do NOT add the boss to any hero array -- it is NOT a Hero.
- Do NOT emit HERO_KILLED from die() -- this is the critical pitfall from research.
