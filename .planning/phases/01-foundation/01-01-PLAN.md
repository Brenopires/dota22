    2. A typed Events constants object:
    ```typescript
    export const Events = {
      HERO_KILLED:        'hero:killed',
      HERO_RESPAWNED:     'hero:respawned',
      MATCH_STATE_CHANGE: 'match:state_change',
      MATCH_TIMER_TICK:   'match:timer_tick',
      RESPAWN_TICK:       'respawn:tick',
      SCORE_UPDATED:      'score:updated',
    } as const;
    ```
    3. A derived type: `export type EventKey = typeof Events[keyof typeof Events];`

    Do NOT use `game.events` — the standalone singleton survives scene restarts cleanly. Do NOT use EventEmitter3 directly — use `Phaser.Events.EventEmitter` (EventEmitter3 is already bundled in Phaser 3.90.0).

    The cleanup pattern for any class that subscribes:
    - Subscribe: `EventBus.on(Events.HERO_KILLED, this.handler, this)` — the third arg `this` is the context scope
    - Unsubscribe: `EventBus.off(Events.HERO_KILLED, this.handler, this)` — must match scope exactly

    Do NOT call `EventBus.removeAllListeners()` in any cleanup path — that removes ALL global listeners, not just the caller's.
  </action>
