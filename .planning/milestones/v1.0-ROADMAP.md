# Milestone v1.0: MVP

**Status:** SHIPPED 2026-02-23
**Phases:** 1-8
**Total Plans:** 41

## Overview

Rift Clash evolves from a working arena brawler into a competitive asymmetric hero brawler with 5-minute matches, a boss economy, team size randomization, and battle modifiers. The build order follows hard dependencies: the event bus and state machine unblock everything; respawn replaces the current instant-defeat behavior; hero identity (ultimates, passives, XP) and team balancing ship before objectives; the boss and tower systems share the same phase because towers require BaseEntity too; battle traits reuse the combat event hooks built for passives; neutral camps finalize the map; Sudden Death and full scoring close the match loop; draft UI and ranked tiers polish the surrounding flow.

## Milestone Summary

**Key Decisions:**
- EventBus singleton using Phaser.Events.EventEmitter (no new deps)
- BaseEntity abstract class with Hero, BossEntity, TowerEntity, NeutralMob extending it
- Forward-only state machine: PRE_MATCH -> ACTIVE -> SUDDEN_DEATH -> ENDED
- Boss Tier 2 roaming uses waypoint graph (not NavMesh) with Phaser Arcade collider sliding
- Flat +/-40 MMR replaces ELO K=32 formula
- Two-step draft: generatePartialMatch() -> player picks -> finalizeMatch()

**Issues Resolved:**
- Scene restart memory leak (tickTimer accumulation) — fixed via MatchStateMachine.destroy()
- Physics body zeroed on die() requiring setCircle() restore on respawn
- Sudden Death race condition at 5:00 boundary — forward-only FSM + endingMatch guard
- Duplicate getRank() in MenuScene/ResultScene — extracted to shared RankUtils.ts

**Issues Deferred:**
- None critical. Stale TODO comment in types.ts (line 205) is cosmetic.

**Technical Debt Incurred:**
- HUD buff icons destroy+recreate each frame (max 4 objects) — acceptable for now
- Score breakdown uses single-color tint (full per-segment coloring needs rich text plugin)
